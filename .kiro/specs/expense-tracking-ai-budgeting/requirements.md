# Requirements Document

## Introduction

M-Expense Flow is a comprehensive gamified financial tracking application specifically designed for Ghanaian university students. The system combines expense tracking, AI-powered budgeting insights, intelligent overspending detection using machine learning, gamification elements, and business management tools to help students understand their spending patterns, build better money habits, and manage both personal finances and side hustles. The application features a freemium model with Ghana-specific categories, ML-powered anomaly detection, and student-focused recommendations.

## Glossary

- **User**: A Ghanaian university student who has registered with the M-Expense Flow application
- **Transaction**: A financial record representing either an Expense or Income entry
- **Expense**: Money spent by the User, categorized into 25 Ghana-specific categories
- **Income**: Money received by the User, categorized into 4 income types
- **Category**: A classification label for Expenses (e.g., Uber/Bolt, Trotro & Transport, Situationship Spending)
- **Quick Budget**: A simple spending limit set by the User for a specific Category and time period
- **Savings Goal**: A target amount the User wants to save for a specific purpose with a deadline
- **Comprehensive Budget**: A complete monthly/semester budget with income allocation across all categories
- **AI Engine**: The intelligent system that analyzes spending patterns and generates student-specific recommendations
- **XP**: Experience Points earned through app usage and financial behaviors
- **Achievement**: A gamification reward unlocked by completing specific financial milestones
- **Streak**: Consecutive days of app usage for habit building
- **Challenge**: Daily, weekly, or monthly tasks that reward XP when completed
- **Mission**: Multi-step quests that guide Users through financial learning journeys
- **Leaderboard**: Rankings of students based on XP earned in different time periods
- **Premium User**: A User who has subscribed to the paid tier with advanced features
- **Business Mode**: Premium feature for tracking business income and expenses separately
- **Overspending Tracker**: Machine learning system that monitors spending patterns in a specific category and alerts users to anomalies
- **Anomaly Detection**: Statistical analysis using Z-score to identify unusual spending behavior (daily and weekly patterns)
- **Category Classification**: Automatic categorization of expenses as Want (discretionary), Need (essential), or Hybrid (mixed purpose)
- **Training Period**: Initial 7-day data collection phase required before anomaly detection becomes active
- **AI Insight**: Personalized financial recommendation generated by the AI Engine
- **Comparative Analysis**: AI feature comparing User's spending to peer averages

## Requirements

### Requirement 1

**User Story:** As a new student, I want to create an account with my name and email, so that I can start tracking my finances and earning XP.

#### Acceptance Criteria

1. WHEN a User provides a valid name and email THEN the System SHALL create a new User profile with default settings (Level 1, 0 XP, 0 streak)
2. WHEN account creation succeeds THEN the System SHALL award the "First Step" achievement and navigate to the Dashboard
3. WHEN a User provides an empty name or invalid email THEN the System SHALL reject registration and display validation errors
4. WHEN a User profile is created THEN the System SHALL initialize gamification data (achievements array, challenge progress)
5. WHEN a new User joins THEN the System SHALL set currency to GHS and darkMode to false by default

### Requirement 2

**User Story:** As a user, I want to log transactions to track my spending and earning, so that I can understand my financial patterns and earn XP.

#### Acceptance Criteria

1. WHEN a User creates a Transaction with valid data THEN the System SHALL store it with a unique timestamp-based ID and award 5 XP
2. WHEN a User creates an Expense THEN the System SHALL require amount, category, and date fields
3. WHEN a User creates an Income THEN the System SHALL require amount, source, and date fields  
4. WHEN a Transaction amount is zero or negative THEN the System SHALL reject creation and display validation error
5. WHEN a Transaction is created THEN the System SHALL check for achievement unlocks and update daily streak

### Requirement 3

**User Story:** As a student, I want to categorize my expenses using Ghana-specific categories, so that I can track spending that's relevant to my lifestyle.

#### Acceptance Criteria

1. WHEN a User creates an Expense THEN the System SHALL allow selection from 25 predefined categories including Uber/Bolt, Trotro & Transport, Situationship Spending, and Impulse TikTok Buys
2. WHEN a User creates an Income THEN the System SHALL allow selection from 4 income categories: Allowance, Part-time Job, Business Income, Other Income
3. WHEN displaying Transactions THEN the System SHALL show category badges with appropriate icons and colors
4. WHEN filtering Transactions THEN the System SHALL support filtering by any category
5. WHEN calculating analytics THEN the System SHALL group spending data by category for insights

### Requirement 4

**User Story:** As a user, I want to create Quick Budgets for specific categories, so that I can control my spending and track progress.

#### Acceptance Criteria

1. WHEN a Free User creates Quick Budgets THEN the System SHALL limit them to maximum 3 budgets
2. WHEN a Premium User creates Quick Budgets THEN the System SHALL allow unlimited budgets
3. WHEN a User creates a Quick Budget THEN the System SHALL require category, amount limit, and period (monthly/weekly/semester)
4. WHEN a User views Budget progress THEN the System SHALL display spent amount, limit, percentage used, and visual progress bar
5. WHEN Budget spending reaches 80% THEN the System SHALL display warning indicator; when exceeding 100% SHALL display alert indicator
6. WHEN Budget progress calculation encounters date range filtering issues THEN the System SHALL use fallback calculation that queries all transactions in the category to ensure accurate progress display

### Requirement 5

**User Story:** As a user, I want to create Savings Goals to work toward financial targets, so that I can build better saving habits.

#### Acceptance Criteria

1. WHEN a Free User creates Savings Goals THEN the System SHALL limit them to maximum 3 goals
2. WHEN a Premium User creates Savings Goals THEN the System SHALL allow unlimited goals
3. WHEN a User creates a Savings Goal THEN the System SHALL require name, target amount, deadline, and category
4. WHEN a User adds contributions to a Goal THEN the System SHALL update current amount and recalculate progress percentage
5. WHEN a Savings Goal is completed THEN the System SHALL award the "Goal Achiever" achievement and appropriate XP

### Requirement 6

**User Story:** As a Premium user, I want to create Comprehensive Budgets, so that I can allocate my entire income across categories and savings.

#### Acceptance Criteria

1. WHEN a Free User attempts Comprehensive Budget THEN the System SHALL display Premium upgrade prompt
2. WHEN a Premium User creates Comprehensive Budget THEN the System SHALL allow setting total income and savings percentage
3. WHEN allocating budget categories THEN the System SHALL ensure total allocation equals 100% of available income
4. WHEN a Comprehensive Budget is active THEN the System SHALL display pie chart visualization and category breakdowns
5. WHEN budget period ends THEN the System SHALL generate performance summary comparing actual vs planned spending

### Requirement 7

**User Story:** As a user, I want to earn XP and unlock achievements, so that budgeting becomes engaging and rewarding.

#### Acceptance Criteria

1. WHEN a User logs a Transaction THEN the System SHALL award 5 XP and check for transaction-based achievements
2. WHEN a User creates a Budget THEN the System SHALL award 15 XP and check for budget achievements
3. WHEN a User creates a Savings Goal THEN the System SHALL award 20 XP and check for savings achievements
4. WHEN XP increases THEN the System SHALL calculate new level using formula: Level = (Total XP ÷ 100) + 1
5. WHEN a User unlocks an Achievement THEN the System SHALL award bonus XP (25-100 based on rarity) and display celebration

### Requirement 8

**User Story:** As a user, I want to maintain daily streaks, so that I build consistent financial tracking habits.

#### Acceptance Criteria

1. WHEN a User opens the app daily THEN the System SHALL increment streak counter and display streak milestone
2. WHEN a User misses a day THEN the System SHALL reset streak to 1 on next app open
3. WHEN streak reaches milestones (7, 14, 30, 50, 100, 365 days) THEN the System SHALL unlock streak achievements and award bonus XP
4. WHEN displaying streak THEN the System SHALL show current count with fire emoji and motivational message
5. WHEN streak is broken THEN the System SHALL offer encouragement to rebuild without penalty

### Requirement 9

**User Story:** As a user, I want to complete daily, weekly, and monthly challenges, so that I can earn extra XP and learn good financial habits.

#### Acceptance Criteria

1. WHEN challenges reset (daily at midnight, weekly on Sunday, monthly on 1st) THEN the System SHALL generate new challenges appropriate to User's level
2. WHEN a User completes a challenge THEN the System SHALL award specified XP (20-200 based on difficulty) and mark as complete
3. WHEN displaying challenges THEN the System SHALL show progress bars, XP rewards, and time remaining
4. WHEN all daily challenges are complete THEN the System SHALL display congratulations and bonus XP opportunity
5. WHEN User consistently completes challenges THEN the System SHALL unlock challenge-specific achievements

### Requirement 10

**User Story:** As a user, I want to complete multi-step missions, so that I can learn comprehensive financial management through guided journeys.

#### Acceptance Criteria

1. WHEN a User starts a Mission THEN the System SHALL display all steps with completion status and overall progress
2. WHEN a Mission step is completed THEN the System SHALL automatically check completion and advance to next step
3. WHEN a Mission is fully completed THEN the System SHALL award large XP bonus (100-300) and exclusive badge
4. WHEN displaying Missions THEN the System SHALL show step-by-step progress, requirements, and rewards
5. WHEN User completes all available Missions THEN the System SHALL unlock "Mission Master" achievement

### Requirement 11

**User Story:** As a student, I want to compete on leaderboards, so that I can compare my financial progress with peers and stay motivated.

#### Acceptance Criteria

1. WHEN displaying Weekly Leaderboard THEN the System SHALL show top 100 students ranked by weekly XP earned
2. WHEN displaying Monthly Leaderboard THEN the System SHALL show top 500 students ranked by monthly XP earned  
3. WHEN displaying All-Time Leaderboard THEN the System SHALL show top 1000 students ranked by total XP
4. WHEN a User views leaderboards THEN the System SHALL highlight their position with "YOU" badge even if ranked below visible range
5. WHEN leaderboard periods reset THEN the System SHALL archive previous rankings and start fresh counters

### Requirement 12

**User Story:** As a user, I want AI-powered spending insights, so that I can understand my financial patterns and receive personalized advice.

#### Acceptance Criteria

1. WHEN the AI Engine analyzes spending THEN the System SHALL generate 8-10 personalized recommendations based on User's patterns
2. WHEN AI detects high spending in a category (>30% of total) THEN the System SHALL provide specific reduction suggestions with local context
3. WHEN AI identifies frequent small transactions THEN the System SHALL calculate semester cost projections and suggest awareness strategies
4. WHEN generating insights THEN the System SHALL use student-friendly language and Ghana-specific advice (trotro vs Uber, midnight data bundles)
5. WHEN Free Users view insights THEN the System SHALL show basic recommendations; Premium Users SHALL see comparative analysis and advanced patterns

### Requirement 13

**User Story:** As a Premium user, I want comparative spending analysis, so that I can see how my spending compares to other students.

#### Acceptance Criteria

1. WHEN Premium User requests comparative analysis THEN the System SHALL compare their spending percentages to anonymized peer averages
2. WHEN displaying comparisons THEN the System SHALL show category-by-category differences (e.g., "You spend 35% on transport vs average 15%")
3. WHEN generating peer comparisons THEN the System SHALL use aggregated anonymous data to protect privacy
4. WHEN spending significantly differs from peers THEN the System SHALL provide context-aware suggestions for optimization
5. WHEN Free Users attempt comparative analysis THEN the System SHALL display Premium upgrade prompt with benefits

### Requirement 14

**User Story:** As a Premium user, I want Business Mode for tracking side hustles, so that I can separate personal and business finances.

#### Acceptance Criteria

1. WHEN Free Users attempt Business Mode THEN the System SHALL display Premium paywall with upgrade options
2. WHEN Premium Users enable Business Mode THEN the System SHALL provide separate transaction tracking for business income and expenses
3. WHEN creating business budgets THEN the System SHALL offer 6 pre-built templates (Fashion Reseller, Food Vendor, Digital Services, Hair & Beauty, Tutoring, E-commerce)
4. WHEN Business Mode is active THEN the System SHALL calculate profit margins, cash flow, and business-specific insights
5. WHEN viewing business dashboard THEN the System SHALL display monthly revenue, expenses, net profit, and profit margin percentage

### Requirement 15

**User Story:** As a student, I want intelligent overspending detection for specific categories, so that I can become aware of spending patterns I don't normally notice and learn better financial habits.

#### Acceptance Criteria

1. WHEN a User selects a category to track THEN the System SHALL create an overspending tracker with hybrid baseline initialization
2. WHEN creating a tracker THEN the System SHALL present Ghana-specific reasonable spending amounts for that category and allow User adjustment
3. WHEN a User confirms baseline amounts THEN the System SHALL use these as initial thresholds for anomaly detection
4. WHEN a Free User attempts to create more than 1 tracker THEN the System SHALL display Premium upgrade requirement
5. WHEN a Premium User creates trackers THEN the System SHALL allow unlimited category tracking
6. WHEN a tracker has less than 21 days of data THEN the System SHALL use baseline thresholds with conservative anomaly detection (3+ std dev only)
7. WHEN a tracker reaches 14 days of data THEN the System SHALL ask User to confirm if their spending patterns feel "typical" or "unusual"
8. WHEN a User marks patterns as "unusual" THEN the System SHALL extend conservative period to 28 days and suggest pattern review
9. WHEN a tracker has 21+ days of typical patterns THEN the System SHALL activate gradual ML integration (10% ML weight week 4, 25% week 5, 50% week 6, 75% week 7, 100% week 8)
10. WHEN ML detects consistent overspending in first 3 weeks THEN the System SHALL delay ML learning and suggest User review spending habits
11. WHEN a tracker has 6+ weeks of data THEN the System SHALL activate weekly trend analysis for gradual overspending patterns
12. WHEN spending anomalies are detected THEN the System SHALL display severity-based alerts (Medium: 2.5+ std dev from adjusted baseline, High: 3+ std dev)
13. WHEN displaying alerts THEN the System SHALL provide different insights for Want categories vs Need categories vs Hybrid categories
14. WHEN ML suggests baseline increases THEN the System SHALL require User confirmation and show spending trend explanation before adjusting thresholds upward
15. WHEN baseline increases exceed 30% from original THEN the System SHALL require User to re-confirm if current spending feels sustainable
16. WHEN a User deletes a tracker THEN the System SHALL allow creating a new tracker for a different category (free users can switch)

### Requirement 16

**User Story:** As a user, I want to use the Bill Split Calculator, so that I can fairly divide expenses with friends and track my portion.

#### Acceptance Criteria

1. WHEN a User enters bill details THEN the System SHALL calculate equal splits or allow custom amounts per person
2. WHEN split is calculated THEN the System SHALL display each person's share and provide copy/share functionality
3. WHEN User completes splits THEN the System SHALL count toward "Split Master" achievements
4. WHEN saving split results THEN the System SHALL offer to add User's portion as a Transaction
5. WHEN using calculator frequently THEN the System SHALL unlock split-related achievements and award XP

### Requirement 17

**User Story:** As a user, I want to view my Dashboard, so that I can quickly understand my financial status and recent activity.

#### Acceptance Criteria

1. WHEN a User accesses Dashboard THEN the System SHALL display current balance (Income - Expenses), monthly spending chart, and top 5 categories
2. WHEN Dashboard loads THEN the System SHALL show streak count, level progress, recent achievements, and quick action buttons
3. WHEN displaying budget health THEN the System SHALL show progress bars for active budgets with color coding (green/yellow/red)
4. WHEN showing recent activity THEN the System SHALL display last 5 transactions with category badges
5. WHEN Dashboard renders THEN the System SHALL complete loading within 2 seconds for good user experience

### Requirement 18

**User Story:** As a user, I want to view Transaction History with filtering, so that I can review and manage my past financial records.

#### Acceptance Criteria

1. WHEN a User accesses History THEN the System SHALL display all Transactions sorted by date descending with search and filter options
2. WHEN filtering by date range THEN the System SHALL return only Transactions within specified period (Last 7 days, 30 days, 3 months, All time)
3. WHEN filtering by category THEN the System SHALL return only Transactions matching selected category
4. WHEN searching by description THEN the System SHALL return Transactions containing search terms
5. WHEN history contains many items THEN the System SHALL implement pagination (20 per page) for performance

### Requirement 19

**User Story:** As a user, I want to manage app settings, so that I can customize my experience and account preferences.

#### Acceptance Criteria

1. WHEN a User toggles Dark Mode THEN the System SHALL immediately apply theme and persist preference to storage
2. WHEN a User updates profile information THEN the System SHALL save changes and reflect them throughout the app
3. WHEN a User requests data export (Premium only) THEN the System SHALL generate Excel/CSV files with all Transaction and Budget data
4. WHEN a User requests account deletion THEN the System SHALL show confirmation dialog and permanently remove all data
5. WHEN settings are modified THEN the System SHALL persist changes locally for offline access

### Requirement 20

**User Story:** As a user, I want Premium subscription management, so that I can upgrade, manage, and understand Premium benefits.

#### Acceptance Criteria

1. WHEN Free Users view Premium features THEN the System SHALL display comparison table, pricing (GHS 19/month with 35% student discount), and funny motivational quotes
2. WHEN a User upgrades to Premium THEN the System SHALL unlock all Premium features immediately and award "Premium Member" achievement
3. WHEN displaying Premium benefits THEN the System SHALL highlight unlimited budgets/goals, AI insights, unlimited overspending trackers, Business Mode, and data export
4. WHEN Free Users hit limits (3 budgets/goals, 1 overspending tracker) THEN the System SHALL show upgrade prompts with specific benefits
5. WHEN Premium subscription is active THEN the System SHALL display Premium badge and crown icon throughout the app

### Requirement 21

**User Story:** As a user, I want the app to work offline, so that I can track expenses without internet connectivity.

#### Acceptance Criteria

1. WHEN the User loses internet connectivity THEN the System SHALL enable offline mode and cache data locally
2. WHEN the User creates or modifies data offline THEN the System SHALL store changes locally with sync pending indicator
3. WHEN internet connectivity is restored THEN the System SHALL automatically synchronize cached changes
4. WHEN displaying data offline THEN the System SHALL show most recent cached version with offline indicator
5. WHEN sync conflicts occur THEN the System SHALL resolve using last-write-wins strategy and notify User

### Requirement 22

**User Story:** As a user, I want my financial data to be secure and private, so that my personal information remains protected.

#### Acceptance Criteria

1. WHEN storing User data THEN the System SHALL apply security rules restricting access to authenticated Users only
2. WHEN a User accesses data THEN the System SHALL verify User owns the requested information
3. WHEN transmitting data THEN the System SHALL use encryption for all communications
4. WHEN handling sensitive information THEN the System SHALL never store passwords or financial account details locally
5. WHEN User requests data deletion THEN the System SHALL permanently remove all associated data and cannot be recovered

### Requirement 23

**User Story:** As a new user setting up overspending detection, I want robust protection against treating my initial overspending as normal behavior, so that the AI learns healthy spending patterns rather than reinforcing bad habits.

#### Acceptance Criteria

1. WHEN a User creates an overspending tracker THEN the System SHALL provide Ghana-specific baseline suggestions for each category with clear context
2. WHEN displaying baseline suggestions THEN the System SHALL show weekly amounts (e.g., Trotro & Transport: GHS 40-60/week, Food & Dining: GHS 80-120/week) with explanation of typical student spending
3. WHEN a User views baseline suggestions THEN the System SHALL allow adjustment with slider controls and "This seems right for me" confirmation
4. WHEN baselines are set THEN the System SHALL store them as initial anomaly detection thresholds and use only these for first 21 days
5. WHEN a tracker reaches 14 days THEN the System SHALL prompt User: "Do your spending patterns in the last 2 weeks feel typical for you, or were you spending more/less than usual?"
6. WHEN a User marks patterns as "unusual" THEN the System SHALL extend baseline-only period to 28 days and suggest spending pattern review
7. WHEN initial 3 weeks show spending consistently 40%+ above baseline THEN the System SHALL delay ML learning and display "High Spending Alert" with habit review suggestions
8. WHEN ML learning begins THEN the System SHALL integrate gradually (10% ML influence week 4, increasing by 15% each week until 100% at week 8)
9. WHEN System suggests baseline increases based on ML THEN the System SHALL require explicit User approval with trend visualization and sustainability check
10. WHEN baseline increases exceed 30% from original THEN the System SHALL require User to confirm "This spending level feels sustainable for my budget"
11. WHEN a User rejects baseline increases 3+ times THEN the System SHALL maintain conservative thresholds and note User preference for strict detection
12. WHEN displaying category defaults THEN the System SHALL provide context (e.g., "Based on typical Ghana university student spending patterns - adjust if your situation is different")
### Requirement 25

**User Story:** As an impatient user, I want the option to activate AI learning faster than the default conservative approach, so that I can get immediate overspending detection even if it's less accurate initially.

#### Acceptance Criteria

1. WHEN creating an overspending tracker THEN the System SHALL offer two setup modes: "Conservative (Recommended)" and "Quick Start"
2. WHEN a User selects Conservative mode THEN the System SHALL follow the 21-day baseline-only approach with gradual ML integration
3. WHEN a User selects Quick Start mode THEN the System SHALL display warning: "AI will learn from your spending immediately. Make sure your first week represents normal spending habits."
4. WHEN Quick Start is confirmed THEN the System SHALL activate ML learning after only 7 days with 50% ML weight, increasing to 100% by day 14
5. WHEN Quick Start detects anomalies in first week THEN the System SHALL show gentler alerts with "Still learning your patterns" disclaimer
6. WHEN Quick Start users spend significantly above baseline in first 2 weeks THEN the System SHALL offer to "Reset to Conservative Mode"
7. WHEN a User switches from Quick Start to Conservative THEN the System SHALL reset learning period and use baseline-only detection for 21 days
8. WHEN displaying mode selection THEN the System SHALL show comparison: "Conservative: More accurate, 3+ weeks to activate" vs "Quick Start: Immediate feedback, may be less accurate initially"
9. WHEN Quick Start users get frequent false alerts THEN the System SHALL suggest switching to Conservative mode after 2 weeks
10. WHEN Quick Start completes 4 weeks successfully THEN the System SHALL congratulate User and note their patterns are well-established

### Requirement 25

**User Story:** As a user with budget calculation issues, I want the system to automatically use fallback calculations when date range filtering fails, so that my budget progress is always displayed accurately regardless of technical issues.

#### Acceptance Criteria

1. WHEN budget progress calculation encounters date range filtering errors THEN the System SHALL automatically switch to fallback calculation without user intervention
2. WHEN fallback calculation is used THEN the System SHALL query all transactions in the category without date filtering to ensure accurate totals
3. WHEN fallback calculation completes THEN the System SHALL return progress data with a 'fallbackUsed' flag for debugging purposes
4. WHEN displaying budget progress using fallback THEN the System SHALL show the same UI elements (spent amount, limit, percentage, alert level) as normal calculation
5. WHEN fallback calculation is successful THEN the System SHALL log the fallback usage for future optimization while maintaining user experience

### Requirement 26

**User Story:** As a user with inconsistent spending habits, I want the AI to distinguish between my intentional spending and impulsive overspending, so that the system helps me build better habits rather than normalizing poor financial decisions.

#### Acceptance Criteria

1. WHEN the System detects spending spikes in first 21 days THEN the System SHALL flag these as "learning period anomalies" and exclude from baseline calculations
2. WHEN ML learning begins THEN the System SHALL weight recent data more heavily than older data to adapt to improving habits
3. WHEN a User consistently spends below baseline for 2+ weeks THEN the System SHALL offer to lower baseline thresholds to match improved habits
4. WHEN baseline adjustments are suggested THEN the System SHALL show spending trend graphs and ask "Does this spending level reflect your intended habits?"
5. WHEN a User reports financial stress or overspending concerns THEN the System SHALL reset to conservative baseline-only detection until patterns stabilize
6. WHEN the System detects weekend vs weekday spending patterns THEN the System SHALL apply different thresholds for different days of the week
7. WHEN semester breaks or exam periods occur THEN the System SHALL allow Users to mark "unusual spending periods" that won't influence ML learning
8. WHEN Users mark spending as "one-time expense" or "emergency" THEN the System SHALL exclude these transactions from pattern learning
9. WHEN ML suggests increases during exam periods or holidays THEN the System SHALL delay suggestions until normal academic periods resume
10. WHEN a User's spending improves consistently for 4+ weeks THEN the System SHALL congratulate progress and offer to lock in improved baselines
